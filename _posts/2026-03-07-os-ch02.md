## 1. 컴퓨터 시스템 구조
![computer_system](/assets\images\ban-hyokyung-os\os_computersystem.png)  
CPU: 계산능력으로 Memory의 instruction을 차례대로 수행한다.<br>  

register: cpu 내부에 있는 저장장치로 cpu가 계산을 할 때 정보를 빠르게 저장하고 가져오는 데 사용된다.  
빠른 순서: register > cash > memory  
저장 크기: register < cash < memory<br>  

interrupt line: cpu가 하던 일을 멈추고 다른 일을 하게 할 때 이것을 사용한다. cpu는 instruction을 하나 수행할 때마다 interrupt line을 확인한다.  
* 시스템콜: 사용자 프로그램이 필요에 의해 호출
* 트랩: 예기치 못한 에러 발생으로 발생
* 하드웨어로 인한 호출: I/O 장치에 의해 호출<br>  

mode bit: 사용자 프로그램이 다른 사용자 프로그램이나 운영체제에 간섭하지 못하게 하는 장치. 보안 문제 해결을 위함도 있다.  
1(사용자모드): 사용자프로그램 수행  
0(모니터모드 ,커널모드, 시스템모드): OS 코드 수행  
ex. 사용자 프로그램이 외부와 상호작용하고 싶으면 운영체제에게 cpu를 넘겨줘서 실행하게 한다.<br>  

timer: cpu를 하나의 사용자 프로그램이 독점하는 걸 막기 위해(time sharing 구현) 일정 시간이 흐른 뒤 운영체제에게 제어권이 넘어가도록 인터럽트를 발생시킴  
매 클럭마다 1씩 줄여듬.<br>  

device controller: I/O 장치를 관리하는 작은 cpu로 I/O 작업이 끝나면 그 데이터값을 local buffer에 저장하고 interrupt로 cpu에게 알림.<br>  

동기식 입출력: I/O 작업이 끝날때까지 프로그램은 대기한다.  
비동기식 입출력: I/O 작업이 진행되는 동안 다른 작업을 계속 수행한다.<br>  

DMA(Direct Memory Access): 외부 장치 인터럽트를 모아 하나의 block 단위로 cpu에게 보낸다.<br>  

서로 다른 입출력 명령어:  
* isolated I/O: 메모리와 I/O 공간이 분리됨. I/O 전용 명령어 필요하고 별도의 제어 선 필요.
* Memory-Mapped I/O: 메모리 주소 공간 공유. 일반 사용 명령어, 동일한 선 공유, 최신 트렌드.<br>  

저장장치 계층 구조
![memory_structure](/assets\images\ban-hyokyung-os\memory_structure.png)  
위로 올라갈수록 speed 빨라지고 cost 비싸다.초록 부분은 휘발성(volatility) 있고 빨간 부분은 아니다.<br>  

프로그램의 실행(memory load):  
프로그램이 가상메모리에 올려지고 주소 변환을 통해 phisical memory에 지정된다. 용량이 크더라도 현재 작업에 사용할 메모리들만 phisical memory에 올려 사용하면 되고 swap area를 이용해 쓰지 않는 데이터를 보관한다.<br>  

가상메모리 구조  
* code: 컴퓨터가 읽을 수 있는 명령어들로 작성된 데이터
* data: 프로그램 시작부터 끝까지 유지될 전역 변수 데이터
* stack: 함수, 지역 변수 등 유동 데이터<br>  

커널 저장 공간
![kernel](/assets\images\ban-hyokyung-os\kernel.png)
PCB: 운영체제가 프로세스의 상태정보를 저장하기 위한 구조체  
프로세스 a에서 프로세스 b로 넘어갈 때 프로세스 a의 정보를 stack에 저장하고 저장된 그 주소값을 data에 저장한다.<br>  

사용자 프로그램이 사용하는 함수
* 사용자 정의 함수: 자신이 프로그램에서 정의한 함수
* 라이브러리 함수: 정의하지 않고 라이브러리에서 갖다 쓴 함수이고 실행파일에 포함되어있다.
* 커널함수: 운영체제 프로그램의 함수이며 시스템 콜로 사용가능하다.
