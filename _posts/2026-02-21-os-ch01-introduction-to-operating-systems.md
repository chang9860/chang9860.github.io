## 1. 운영체제
- 운영체제란?<br>: 컴퓨터 하드웨어와 모든 사용자 및 소프트웨어 사이를 연결해주는 소프트웨어 계층  
![OS concept](/assets/images/ban-hyokyung-os/os_concept.png)

- 운영체제의 목표  
1. 컴퓨터 시스템의 자원들을 효율적으로 관리하는 것
2. 사용자가 시스템을 편리하게 사용할 수 있도록 하는 것

## 2. 운영체제의 분류
- 동시 작업 가능 여부  
    * 단일 작업
    * 다중 작업
- 사용자의 수  
    * 단일 사용자
    * 다중 사용자
- 처리 방식
    * 일괄 처리<br>: 작업량을 모아놨다가 일정 이상이 되면 한꺼번에 처리하는 방식
    * 시분할(time sharing)<br>: 여러 작업을 실행할 때 일정 시간 단위로 분할하여 돌아가면서 자원을 배당하는 방식
    * 실시간(Realtime OS)<br> 정해진 시간 내로 어떠한 일이 반드시 종료됨이 보장되어야 하는 실시간 시스템을 위한 OS<br> ex. 미사일 제어, 원자로/공장 제어, 반도체 장비, 로보트 제어

## 3. 운영체제의 예시
- Unix<br>: 여러 사용자가 서버로서 이용가능하고 대부분 c언어로 작성되어 있어 가볍고 이식성이 뛰어나다. 오픈소스이며 Linux가 이를 기반으로 만들어졌다.
- Dox<br>: 단일 사용자, 단일 작업만 가능하고 작은 메모리의 한계(주 기억장치 640KB)가 있다.
- MS Windows<br>: MS사의 다중 작업용 GUI 기반 운영체제

## 4. 운영체제 구조 
![운영체제 구조](/assets\images\ban-hyokyung-os\os_structure.png)

- CPU 스케줄링<br>: 여러 프로그램의 cpu요청에 cpu를 어떻게 할당할지
- 메모리 관리<br>: 어떤 프로그램을 메모리에 올릴 것인지를 정한다. 메모리에 올려진 데이터들은 cpu가 바로 다룰 수 있다. 메모리에 없는 건 Disk에 보관되어있다. cpu가 Disk에서 데이터를 꺼낼때 시간이 더 오래 걸린다.
- 파일 관리<br>: 디스크에서 header로 데이터를 꺼낼 때 어떻게 꺼내야 할지. ex. 엘리베이터가 어느 층 먼저 갈지와 유사.
- 입출력 관리<br>: 키보드, 마우스 등 입출력이 들어왔을 때 CPU가 어떻게 이를 관리하는지.
